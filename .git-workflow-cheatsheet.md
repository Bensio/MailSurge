# Git Workflow Cheat Sheet

Quick reference for daily development workflow.

## Current Branch Setup

- `main` → Production (deploys to custom domain)
- `develop` → Development (your working branch)

## Daily Development Workflow

```bash
# 1. Start working (switch to develop)
git checkout develop
git pull origin develop

# 2. Make your changes
# ... edit files ...

# 3. Commit changes
git add .
git commit -m "feat: description of changes"
git push origin develop

# 4. When ready for production
git checkout main
git pull origin main
git merge develop
git push origin main
# Vercel automatically deploys!
```

## Quick Commands

### Switch Branches
```bash
git checkout develop    # Switch to development
git checkout main       # Switch to production
```

### Check Status
```bash
git status              # See current changes
git branch -a           # See all branches
```

### Sync with Remote
```bash
git pull origin develop # Get latest from develop
git pull origin main    # Get latest from main
```

### Deploy to Production
```bash
git checkout main
git merge develop
git push origin main
```

## Feature Branches (Optional)

For larger features:
```bash
# Create feature branch
git checkout develop
git checkout -b feature/feature-name

# Work on feature
# ... make changes ...
git add .
git commit -m "feat: feature description"
git push -u origin feature/feature-name

# Merge back to develop
git checkout develop
git merge feature/feature-name
git push origin develop
```

## Emergency Hotfix

If you need to fix production immediately:
```bash
# Create hotfix from main
git checkout main
git checkout -b hotfix/critical-bug

# Fix the bug
# ... make changes ...
git add .
git commit -m "fix: critical bug description"
git checkout main
git merge hotfix/critical-bug
git push origin main

# Also merge to develop
git checkout develop
git merge hotfix/critical-bug
git push origin develop
```

## Current Status

Run this to see where you are:
```bash
git status
git branch
```

## Vercel Configuration

Remember to configure Vercel:
1. Go to Vercel Dashboard → Your Project → Settings → Git
2. Set **Production Branch** to `main`
3. (Optional) Enable preview deployments for other branches
